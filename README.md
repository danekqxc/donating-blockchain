# 💎 Donation DAO | Invest Portal

Децентрализованное приложение (DApp) для управления пожертвованиями и инвестирования в проекты внутри экосистемы. Пользователи могут жертвовать ETH, получать взамен токены управления (DP) и распределять их между различными целями.

## 🚀 Основные функции
- **Minting System:** Обмен ETH на токены `Donation Points (DP)` по курсу 1 ETH = 10,000 DP.
- **DAO Investing:** Распределение полученных очков между пулами (Маркетинг, Разработка, Благотворительность).
- **Withdraw System:** Возможность забрать свой вклад обратно в личный баланс очков.
- **Web3 Interface:** Современный UI с интеграцией MetaMask и Ethers.js.

## 🛠 Технологический стек
- **Smart Contracts:** Solidity 0.8.0 (ERC20-like logic).
- **Frontend:** HTML5, CSS3 (Modern Glassmorphism Design).
- **Blockchain Interaction:** Ethers.js.
- **Tools:** Remix IDE, MetaMask.

## 📦 Структура проекта
- `DonationToken.sol` — Смарт-контракт на Solidity. Реализует логику минтинга, хранения инвестиций и управления балансом.
- `index.html` — Фронтенд-интерфейс с использованием CSS-переменных и Flexbox/Grid.
- `app.js` — Логика взаимодействия с блокчейном: подключение кошелька, вызов функций контракта и обновление UI в реальном времени.

## ⚙️ Как запустить проект локально

1. **Развертывание контракта:**
   - Откройте `DonationToken.sol` в [Remix IDE](https://remix.ethereum.org/).
   - Разверните контракт в тестовой сети (например, Sepolia).
   - Скопируйте адрес развернутого контракта.

2. **Настройка фронтенда:**
   - Откройте файл `app.js`.
   - Вставьте адрес вашего контракта в переменную `contractAddress`:
     ```javascript
     const contractAddress = "ВАШ_АДРЕС_КОНТРАКТА";
     ```

3. **Запуск:**
   - Откройте `index.html` в браузере с установленным расширением MetaMask.
   - Подключите кошелек к нужной тестовой сети.

## 📊 Математика контракта
- Минимальный донат: `0.001 ETH`.
- Формула минта: `msg.value * 10000`.
- Все инвестиции внутри системы работают на 18-значных числах (decimals), что обеспечивает высокую точность расчетов.

## 🛡 Безопасность
- Использованы модификаторы доступа для критических функций (вывод ETH владельцем).
- Проверки `require` на достаточность баланса перед инвестированием.
- Безопасный перевод средств через `call`.
